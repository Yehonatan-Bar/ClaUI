שחזור אוטומטי של סשני ClaUi בפתיחה מחדש של VS Code

תאריך: 16 באפריל 2026
סטטוס: הושלם ונפרס
גרסה: 0.1.112
סוג שינוי: פיצ'ר חדש
השפעה על משתמשים קיימים: אפס (מושבת כברירת מחדל)
רמת מורכבות: בינונית

במשפט אחד

הוספנו יכולת שמזכרת את הסשנים שהיו פתוחים לפני שסגרת את VS Code, ופותחת אותם מחדש אוטומטית בפעם הבאה שתפתח את אותו פרויקט.

הבעיה שנפתרה

עד היום, כשמשתמש סגר את VS Code וחזר אליו, הוא היה צריך לפתוח כל סשן מחדש ידנית דרך היסטוריית השיחות. אם היו לו חמש שיחות פתוחות עם Claude ו-Codex - כולן נעלמו בסגירה, והיה עליו לזכור אילו שיחות היו פעילות ולהחיות כל אחת בנפרד.

הפיצ'ר החדש שומר בשקט, ברקע, תמונת מצב של הסשנים הפתוחים - וברגע שהמשתמש מפעיל את ההגדרה, הפתיחה מחדש של הפרויקט ב-VS Code תחזיר את כל הסשנים אוטומטית, עם התוכן ושם הטאב שהמשתמש נתן להם.

איך זה עובד מבחינת המשתמש

  1. המשתמש נכנס להגדרות VS Code ומחפש "ClaUi Restore Sessions On Startup".
  2. מפעיל את המתג (כברירת מחדל כבוי).
  3. ממשיך לעבוד רגיל - פותח שיחות, נותן להן שמות, סוגר ופותח.
  4. בפעם הבאה שהפרויקט נפתח: הודעת התקדמות מופיעה ("משחזר N סשנים"), הסשנים נפתחים בזה אחר זה בסדר המקורי, עם הצבעים והשמות המקוריים, ועם ההיסטוריה המלאה של השיחה.

מה מאחורי הקלעים

VS Code מספק אחסון מתמיד לכל פרויקט. אנחנו משתמשים בו כדי לשמור רשומה קטנה עם פרטי כל סשן פתוח: מספר הטאב, סוג הספק (Claude / Codex / Happy), מזהה הסשן הקבוע, שם מותאם אישית אם יש.

כשהסשן נכנס לפעולה או משתנה שמו - הרשומה מתעדכנת בלוח זמנים מרוכך (500ms) כדי לא להכביד על הדיסק אבל כן לשרוד קריסות. בסגירה רגילה של VS Code יש "שטיפה" סופית סינכרונית לדיסק.

בפתיחה מחדש: אם ההגדרה מופעלת, המערכת קוראת את התמונה, מסננת רשומות לא תקינות (סשנים בלי מזהה, פרויקט אחר, כפילויות), ופותחת את הטאבים אחד אחרי השני עם מנגנון "--resume" של ה-CLI. אם סשן מסוים נכשל (כי הקובץ נמחק וכו') - הוא מדלג עליו ומציג הודעה מסכמת בסוף.

יתרונות עיקריים

  • חיסכון זמן: המשתמש לא משחזר ידנית שיחות אחרי כל סגירה
  • שמירת הקשר: ההיסטוריה, השם, הצבע - הכל חוזר
  • בטוח: מושבת כברירת מחדל, אף אחד לא יופתע
  • עמיד לקריסות: שמירה דחוסה גם אם VS Code נסגר בפתאומיות
  • ניתן להשבתה מיידית: כיבוי ההגדרה - חוזרים להתנהגות הישנה

סיכונים ומגבלות

  • תלות במזהי CLI: אם מחיקת JSONL או עדכון CLI ישבור את --resume, שחזור יכשל לסשנים מסוימים. יש הודעה ברורה למשתמש.
  • חלון ראשון בלבד: VS Code עם כמה חלונות של אותו פרויקט - רק הראשון שיפתח יקבל את השחזור.
  • זמן אתחול: שחזור סדרתי של 10 טאבים לוקח 10-20 שניות. לא ניתן להקטין בגלל תחרות על ה-CLI.
  • תמונת מצב מוגבלת: אם משתמש ישנה את הפרויקט בידיים (Git checkout וכו') - הטאבים עדיין יחזרו כי הם תלויים בפרויקט, לא בקוד.

איך מפעילים

  1. בתפריט File → Preferences → Settings
  2. מחפשים: restoreSessionsOnStartup
  3. מסמנים את תיבת הסימון
  4. סוגרים ופותחים את VS Code - וזהו

איך מוודאים שהכל עובד

  1. פתיחת שלושה טאבי ClaUi (שני Claude + אחד Codex), שליחת הודעה בכל אחד, שינוי שם לטאב אחד
  2. סגירת VS Code בצורה רגילה
  3. פתיחה מחדש של אותו פרויקט
  4. ציפייה: הודעת התקדמות, שלושת הטאבים נפתחים לפי הסדר, עם השמות והצבעים המקוריים
  5. מבחן קריסה: הרג VS Code דרך Task Manager באמצע סשן - לאחר פתיחה מחדש הטאבים עדיין חוזרים
  6. מבחן פרויקט אחר: פותחים פרויקט B, וידוא שלא נפתחים טאבים של פרויקט A
  7. מבחן סשן שבור: מחיקת קובץ JSONL ידנית, פתיחה מחדש - הודעה "שוחזרו X מתוך Y" תופיע
שימו לב: הפיצ'ר אינו נוגע בשום חלק אחר של התוסף. משתמשים שלא ידליקו את ההגדרה יקבלו את אותה חוויה שהיתה להם קודם, ללא שינוי. זאת סיבה שבחרנו ברירת מחדל "כבוי" - זה פיצ'ר תוספתי לחלוטין.
