### 🟢 Prompt Caching — חיסכון עד 90%



ישנן שתי אפשרויות לכתיבת קאש:

| פעולה | מכפיל מחיר | משך תוקף |
|---|---|---|
| כתיבת קאש (5 דקות) | **×1.25** מחיר input רגיל | 5 דקות |
| כתיבת קאש (1 שעה) | **×2.0** מחיר input רגיל | שעה אחת |
| **קריאה מקאש (cache hit)** | **×0.1** מחיר input רגיל | כמשך הכתיבה |



כלומר, **קריאה מקאש עולה רק 10% מהמחיר הרגיל.** הנה מה שזה אומר בפועל לפי מודל:

| מודל | Input רגיל | Cache Write (5m) | Cache Write (1h) | Cache Read |
|---|---|---|---|---|
| Opus 4.8/4.7 | $5.00 | $6.25 | $10.00 | **$0.50** |
| Sonnet 4.6 | $3.00 | $3.75 | $6.00 | **$0.30** |
| Haiku 4.5 | $1.00 | $1.25 | $2.00 | **$0.10** |

הקאש משתלם כבר אחרי קריאה אחת בקאש 5 דקות (1.25x כתיבה), ואחרי שתי קריאות בקאש שעה (2x כתיבה).

### 🛠️ תמחור כלים (Tools)

כלי web search עולה **$10 לכל 1,000 חיפושים**, בנוסף לעלות הטוקנים הרגילה. כלי web fetch אינו גובה תוספת מחיר מעבר לטוקנים. Code execution חינמי כשמשולב עם web search/web fetch; ללא שילוב, מחיר $0.05 לשעה לקונטיינר, עם **1,550 שעות חינמיות** לחודש לארגון.

### 🤖 Claude Managed Agents — תמחור מיוחד

Managed Agents מחויב בשני מימדים: טוקנים (לפי מחיר רגיל/קאש) + **$0.08 לשעת ריצה** (session-hour). זמן idle אינו נחשב.

### 💡 אסטרטגיית חיסכון מקסימלית

דוגמה מעשית: אפליקציה עם 10,000 בקשות ביום, 2,000 טוקני input ו-500 output, Sonnet 4.6 — עלות בסיס: ~$4,050/חודש. אם 80% מה-input ניתן לקאש, העלות יורדת ל-~$1,500-2,000/חודש.

**שלושת הכלים המרכזיים לחיסכון:**
- **Prompt Caching** — עד 90% חיסכון על input שחוזר על עצמו
- **Batch API** — 50% הנחה אוטומטית לעבודות לא-דחופות
- **ניתוב חכם** — Haiku לפעולות פשוטות, Sonnet לרוב העבודה, Opus רק לצרכים מורכבים
