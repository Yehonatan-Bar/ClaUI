תוכנית: ניהול מפתח API ואבטחת סביבת עבודה

רקע הבעיה

משתמש במכונה אחרת קיבל שגיאת "Credit balance is too low" למרות שיש לו מנוי פעיל של Claude.

הסיבה: התוסף מעביר את כל משתני הסביבה של המערכת לתהליכי Claude. אם יש משתנה ANTHROPIC_API_KEY בסביבה (למשל מפרויקט אחר), Claude CLI משתמש בו במקום באימות המנוי הרגיל - ומגלה שאין קרדיט במפתח הזה.

מה נעשה

1 חסימת ירושת מפתח API מהמערכת

ניצור כלי מרכזי שכל התהליכים ישתמשו בו. הכלי ימחק אוטומטית כל מפתח API שמגיע מהסביבה הכללית, כך שלא יהיה שימוש מקרי במפתח שלא נועד לתוסף.

תוצאה: 9 נקודות הרצה של Claude CLI ישתמשו בכלי המרכזי, במקום קוד כפול בכל מקום.

2 אפשרות להזנת מפתח API בהגדרות

בלוח ההגדרות (כפתור הגלגל ליד Vitals) נוסיף שורה חדשה: API Key.

  • כשאין מפתח מוגדר - כפתור "Set" שפותח שדה הזנה מוסתר (סיסמה)
  • כשיש מפתח - מציג את 4 התווים האחרונים בלבד (למשל ****abcd) + כפתור "Clear"
  • המפתח נשמר בצורה מאובטחת דרך מחזיק המפתחות של מערכת ההפעלה (Windows Credential Manager)
3 חיבור המפתח להרצת התהליכים

כשמשתמש מגדיר מפתח בהגדרות, הוא יוזרק לכל תהליכי Claude CLI. כשאין מפתח מוגדר - שום מפתח לא יעבור (גם לא מהסביבה).

סיכום שינויים

אזור מה משתנה סוג
כלי סביבה מרכזי קובץ חדש שמנקה ומכין משתני סביבה חדש
9 מנהלי תהליכים מעבר לשימוש בכלי המרכזי עריכה
ממשק הגדרות שורת הזנת מפתח API חדשה עריכה
תקשורת תוסף-ממשק הודעות חדשות להגדרה וקריאת מצב המפתח עריכה
מנהלי הודעות טיפול בהגדרת מפתח + אחסון מאובטח עריכה

אימות

1. בדיקה שהממשק מציג נכון את שורת ה-API Key בהגדרות
2. הזנת מפתח - המפתח נשמר ומוצג מוסתר (4 תווים אחרונים)
3. ניקוי מפתח - חזרה למצב "Set"
4. הרצת סשן ללא מפתח - אין ANTHROPIC_API_KEY בסביבה
5. הרצת סשן עם מפתח - ANTHROPIC_API_KEY מוזרק לסביבה
6. במכונה עם ANTHROPIC_API_KEY בסביבה - המפתח נחסם אם לא מוגדר בהגדרות

השפעה

פתרון מיידי: מניעת השגיאה "Credit balance is too low" עבור משתמשים שיש להם מפתח API בסביבה מסיבות אחרות.

יכולת חדשה: משתמשים שרוצים להשתמש במפתח API שלהם (במקום מנוי) יכולים להגדיר אותו בקלות דרך ממשק ההגדרות.
