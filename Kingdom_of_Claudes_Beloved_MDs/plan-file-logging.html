תוכנית: שמירת לוגים לקבצים

רקע

כרגע כל הלוגים של התוסף מוצגים רק בחלון Output של VS Code. הבעיה: כשסוגרים את VS Code או טוענים מחדש - הלוגים נמחקים. התוכנית מוסיפה שמירה אוטומטית של הלוגים לקבצים בדיסק, כך שהם נשמרים לצמיתות.

מה ישתנה?

קובץ לוג לכל סשן (טאב)

כל טאב שנפתח יקבל קובץ לוג משלו. שם הקובץ יכלול את שם הסשן ואת התאריך והשעה.

דוגמה: Fix_login_bug_18-14-30.log

שם הקובץ מתעדכן אוטומטית

כשהמערכת נותנת שם אוטומטי לסשן (או כשהמשתמש משנה שם ידנית) - קובץ הלוג משתנה בהתאם.

קובץ חדש נוצר כאשר:
  • נשלחת פקודת Developer: Reload Window - כי כל טעינה מחדש מפעילה את התוסף מאפס
  • הקובץ גדול מ-2MB - רוטציה אוטומטית
  • נפתח סשן חדש (טאב חדש)
קובץ לוג גלובלי

בנוסף לקבצים של כל טאב, יהיה קובץ לוג מרכזי שמכיל את כל ההודעות מכל הטאבים יחד - שימושי לראיית תמונה כוללת.

הגדרות חדשות

הגדרה תיאור ברירת מחדל
הפעלת לוגים לקובץ מאפשר/מכבה כתיבה לדיסק מופעל
תיקיית לוגים אפשרות לשנות את מיקום שמירת הלוגים תיקיית התוסף ב-VS Code

פקודה חדשה

Claude Mirror: Open Log Directory

פקודה שפותחת את תיקיית הלוגים בסייר הקבצים - לגישה מהירה לקבצי הלוג.

איפה הלוגים נשמרים?

ברירת מחדל: תיקיית האחסון של התוסף ב-VS Code

%APPDATA%/Code/User/globalStorage/claude-code-mirror.../logs/

ניתן לשנות דרך ההגדרות.

שלבי ביצוע

  1. יצירת מודול FileLogger חדש - מנגנון הכתיבה, הרוטציה ושינוי השם
  2. חיבור למנגנון ההפעלה של התוסף - לוגר גלובלי + העברת נתיב התיקייה
  3. חיבור למנהל הטאבים - העברת הנתיב לכל טאב חדש
  4. חיבור לכל סשן (טאב) - לוגר פרטי, עדכון שם, ניקוי משאבים
  5. הוספת הגדרות ופקודה חדשה
  6. בנייה, התקנה ובדיקות

בדיקות

  • פתיחת סשן - וידוא שקובץ לוג נוצר
  • שליחת הודעה - וידוא ששם הקובץ מתעדכן עם שם הסשן
  • שינוי שם ידני - וידוא שהקובץ משתנה שם
  • פתיחת טאב שני - וידוא קובץ נפרד
  • טעינה מחדש של VS Code - וידוא קבצים חדשים
  • הפעלת פקודת פתיחת תיקיית לוגים
