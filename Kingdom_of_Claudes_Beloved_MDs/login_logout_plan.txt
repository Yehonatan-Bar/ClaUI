# Plan: Add Login/Logout to VitalsInfoPanel

## Context

The user wants Login and Logout buttons in the settings gear dropdown (VitalsInfoPanel), mirroring `claude auth login` and `claude auth logout` CLI commands. The Codex login flow pattern already exists as a proven approach (opens a VS Code terminal for interactive auth). The `AuthManager` stub exists but needs real implementation.

## Implementation Steps

### 1. Extend `AuthManager.ts`
**File:** `src/extension/auth/AuthManager.ts`

- Add `AuthStatus` interface: `{ loggedIn: boolean; email: string; subscriptionType: string }`
- Add `getAuthStatus(cliPath: string): Promise<AuthStatus>` - runs `claude auth status --json` via `child_process.execFile`, parses JSON, returns status. Falls back to `{ loggedIn: false, email: '', subscriptionType: '' }` on error. 10s timeout.
- Add `logout(cliPath: string): Promise<boolean>` - runs `claude auth logout` via `execFile`, returns true/false. 10s timeout.
- Export `AuthStatus` interface.

## Key Design Decisions

- **Login via terminal** (not embedded in webview): The `claude auth login` flow is interactive (opens browser, asks for paste). The proven Codex pattern opens a VS Code terminal. No reliable "command completed" detection exists, so a manual Refresh button is provided.
- **Logout via execFile** (not terminal): `claude auth logout` is non-interactive, so it runs silently in the background.
- **Status via execFile**: `claude auth status --json` returns structured JSON, parsed directly.
- **No new dependencies**: Uses only `child_process.execFile` (already available in Node.js).
