מה זה?

כיום הדשבורד מציג נתונים רק של הסשן הנוכחי - כשהסשן נגמר, כל הנתונים נעלמים. הדשבורד החדש יאגור נתונים לאורך כל הסשנים בפרוייקט ויציג תמונת מצב כוללת:

  • כמה עלו כל הסשנים בסך הכל?
  • מה קצב העלויות לאורך זמן?
  • אילו כלים הכי נמצאים בשימוש בפרוייקט הזה?
  • מה התפלגות סוגי המשימות?
  • כמה טוקנים נצרכו בסך הכל?

הדשבורד הנוכחי (ברמת הסשן) נשאר כמו שהוא - אפשר לעבור בין שני המצבים בלחיצה.

איך זה עובד?

1. שכבת שמירה חדשה

בכל פעם שסשן מסתיים (בהצלחה או בקריסה), המערכת שומרת סיכום של הסשן - עלויות, מספר תורות, כלים שהשתמשו בהם, טוקנים, סוגי משימות, ועוד.

הסיכום נשמר במאגר הנתונים של VS Code (workspaceState) שמשויך לפרוייקט הנוכחי. הנתונים שורדים הפעלות מחדש.

2. מתג סשן/פרוייקט בדשבורד

בדשבורד הקיים יתווסף מתג בראש העמוד: Session Project

לחיצה על "Project" מציגה טאבים עם נתונים מצטברים מכל הסשנים. לחיצה על "Session" מחזירה לדשבורד הנוכחי (ללא שינוי).

3. תזרים הנתונים

CLI result event --> MessageHandler builds TurnRecord
                    --> accumulates in extension memory

Session ends --> builds SessionSummary from accumulated data
                --> saves to workspaceState (persistent)

User opens Project tab --> requests data from extension
                         --> renders charts from saved summaries

טאבים בדשבורד הפרוייקט

Overview סקירה כללית

  • כרטיסי מטריקות: סך סשנים, עלות כוללת, סך תורות, עלות ממוצעת לסשן, שיעור שגיאות, סך שימושי כלים, מודל נפוץ, זמן ממוצע
  • גרף מגמת עלויות: עלות לכל סשן לאורך זמן
  • תדירות כלים מצטברת: 15 הכלים הנפוצים ביותר בכל הסשנים
  • התפלגות קטגוריות: תרשים דונאט של כל הקטגוריות מכל הסשנים
  • שימוש במודלים: אילו מודלים שימשו וכמה פעמים

Sessions רשימת סשנים

  • טבלה ממוינת: שם, תאריך, מודל, תורות, עלות, שגיאות, זמן, כלי מוביל
  • לחיצה על שורה מרחיבה פרטים: תדירות כלים, התפלגות קטגוריות, פירוט טוקנים
  • חיפוש טקסט חופשי לפי שם סשן

Tokens טוקנים מצטברים

  • כרטיסי סיכום: קלט, פלט, מטמון יצירה, מטמון קריאה, שיעור מטמון
  • גרף עמודות: פירוט טוקנים לכל סשן

Tools כלים מצטברים

  • תדירות כלים מצטברת מכל הסשנים
  • התפלגות קטגוריות מצטברת
  • התפלגות סוגי משימות (מניתוח סמנטי)

מה נשמר לכל סשן?

סיכום קומפקטי (~500 בתים) כולל:

  • מזהה, שם, מודל, זמני התחלה וסיום
  • עלות כוללת, מספר תורות, שגיאות, שימושי כלים
  • סה"כ טוקנים (קלט, פלט, מטמון)
  • מפת תדירות כלים (שם כלי -> מספר פעמים)
  • התפלגות קטגוריות ומשימות

עד 200 סשנים נשמרים. בנפח של ~100KB - קל מאוד על המערכת.

מה לא משתנה

הדשבורד הקיים (ברמת הסשן) לא נפגע כלל. כל 6 הטאבים הנוכחיים (Overview, Tokens, Tools, Timeline, Commands, Context) ממשיכים לעבוד בדיוק כמו קודם.

ההבדל היחיד: מתג חדש בראש הדשבורד שמאפשר לעבור למצב "Project".

שלבי ביצוע

  1. הוספת טיפוסים - הגדרת מבנה הנתונים של סיכום סשן ופרוטוקול ההודעות
  2. שכבת שמירה - יצירת מאגר נתונים שמנהל סיכומי סשנים ב-workspaceState
  3. צבירת נתונים - אגירת נתוני תורות בצד השרת ושמירת סיכום בסיום סשן
  4. חיווט - העברת נתוני הפרוייקט מהשרת לממשק המשתמש
  5. ממשק - מתג Session/Project + 4 טאבים חדשים עם גרפים
  6. בנייה ובדיקה - בנייה, פריסה, אימות, תיעוד
