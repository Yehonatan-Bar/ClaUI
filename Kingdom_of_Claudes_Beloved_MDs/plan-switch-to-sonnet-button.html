תוכנית: כפתור מעבר מהיר ל-Sonnet 4.6

מטרה: הוספת כפתור ייעודי בסרגל התחתון של התוסף שמאפשר למשתמש לעבור מיידית למודל Sonnet 4.6 באמצע שיחה פעילה, בלי לאבד את ההיסטוריה.

הבעיה

כיום, בורר המודלים בסרגל התחתון משנה את המודל רק עבור השיחה הבאה. אין דרך לשנות את המודל של שיחה שכבר רצה. המשתמש צריך כפתור שמבצע את השינוי מיידית.

הפתרון

כפתור כחול קטן בסרגל התחתון (ליד בורר המודלים) שבלחיצה אחת עוצר את התהליך הנוכחי וממשיך אותו מחדש עם Sonnet 4.6. ההיסטוריה נשמרת במלואה -- המשתמש ממשיך לשוחח כרגיל, רק עם מודל שונה.

איך זה עובד

User clicks [S] button
  -> Extension stops current CLI process
  -> Extension restarts with --resume (same session) + --model sonnet-4.6
  -> CLI picks up where it left off, now using Sonnet
  -> Button auto-hides (already on Sonnet)
  -> User continues chatting normally

שלבי הפיתוח

1
הגדרת הודעה חדשה
הוספת סוג הודעה "switchToSonnet" לחוזה התקשורת בין ממשק המשתמש לתוסף.
2
לוגיקת המעבר בצד השרת
הוספת פונקציה שעוצרת את התהליך הנוכחי וממשיכה אותו מחדש עם המודל החדש. משתמשת במנגנון קיים ומוכח (אותו מנגנון כמו ביטול וחידוש שיחה).
3
טיפול בהודעה
חיבור ההודעה מממשק המשתמש ללוגיקת המעבר בצד השרת.
4
יצירת הכפתור בממשק
רכיב React קטן שמציג כפתור "S" כחול. הכפתור מוסתר כשכבר על Sonnet או כשאין שיחה פעילה, ומושבת כשהמודל עסוק בתשובה.
5
שילוב בסרגל ועיצוב
הצבת הכפתור בסרגל התחתון ליד בורר המודלים, עם עיצוב כחול מודגש שתואם לשפה העיצובית הקיימת.

מצבי קצה

מצב התנהגות
לחיצה בזמן שהמודל עסוק הכפתור מושבת -- יש להמתין או לבטל קודם
אין שיחה פעילה הכפתור מוסתר
כבר על Sonnet 4.6 הכפתור מוסתר
שגיאה בחידוש הצגת הודעת שגיאה למשתמש
לחיצה כפולה מהירה המערכת מטפלת בזה אוטומטית -- תהליך ישן נעצר לפני שחדש מתחיל

היקף השינוי

6 קבצים (1 חדש, 5 עריכות קטנות). השינוי מבוסס על מנגנון קיים ומוכח בקוד, ולכן הסיכון נמוך.

בדיקה

לאחר הפיתוח, יש לוודא:

V
פתיחת שיחה עם מודל שאינו Sonnet -> לחיצה על הכפתור -> וידוא שהמודל הפעיל השתנה ל-Sonnet 4.6 -> וידוא שהשיחה ממשיכה רגיל עם כל ההיסטוריה -> וידוא שהכפתור נעלם אחרי המעבר.
