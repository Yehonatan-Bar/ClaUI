How Documentation Becomes Skills

Every development task generates structured documentation (SR-PTD). Over time, these documents are analyzed, clustered, and transformed into reusable Skills that make AI assistants smarter and more specialized.

The Core Idea

Documentation as Investment, Not Overhead

Traditional documentation is written once and forgotten. Our approach is different: every task a developer completes is documented in a skill-oriented format (called SR-PTD). This documentation isn't just for record-keeping -- it's raw material that gets refined into reusable Skills that teach AI assistants how to handle similar tasks in the future.

Think of it like this: the first time you solve a problem, you document how you solved it. After solving similar problems 3-5 times, those documents are merged into a Skill -- a reusable knowledge package that handles the task automatically next time.

How It Works, Step by Step

1

Task Completion Triggers Documentation

After completing any development task (feature, bug fix, refactor), the developer creates an SR-PTD document. This is mandatory -- no task is considered complete without its documentation.

2

Skill-Oriented Structure

The SR-PTD format is not a generic report. Each section maps directly to a future Skill component:

What's Documented What It Becomes in a Skill
What triggered the task → When should the skill activate?
Steps taken to complete it → Step-by-step instructions
Knowledge needed (APIs, schemas) → Reference materials
Code written during the task → Reusable scripts
Output formats produced → Templates and assets
3

4

Similar Documents Are Clustered

After 3-5 SR-PTD documents accumulate for similar types of tasks, they are grouped together. Common patterns are extracted:

  • What steps appear in every instance?
  • What varies between instances and why?
  • What code was reused or rewritten each time?
  • What knowledge was needed every time?
5

A Reusable Skill Is Built

The clustered patterns are assembled into a formal Skill -- a self-contained package with instructions, scripts, references, and templates.

my-skill/
  SKILL.md ← Core instructions (from workflow patterns)
  scripts/ ← Reusable code (from documented code snippets)
  references/ ← Knowledge base (from documented APIs, schemas)
  assets/ ← Templates (from documented output formats)
6

The Skill Works Automatically

Once a Skill exists, the AI assistant detects when a new task matches the Skill's trigger pattern and activates it automatically. Instead of solving the problem from scratch, the assistant uses the documented best practices, scripts, and references.

Skills also evolve: when a task reveals a gap or improvement, the Skill is updated through a validated change process.

What Makes This Approach Different

Traditional Documentation

Written for humans to read. Sits in a wiki. Gets outdated. Provides background information but doesn't do anything.

Skill-Oriented Documentation (SR-PTD)

Written in a structure that maps to actionable AI capabilities. Accumulates and compounds. After enough examples, it transforms into a Skill that automates the work itself. Every document is an investment that pays dividends.

The Skill Lifecycle

1. Capture

Every task generates an SR-PTD document. This is mandatory and enforced by the development workflow.

2. Accumulate

Documents are stored, tagged, and organized by domain, language, and task pattern for easy clustering.

3. Cluster

When 3-5 similar documents exist, they are grouped. Common workflows, repeated code, and shared knowledge are identified.

4. Extract

A formal Skill is built from the clustered patterns -- with instructions, scripts, reference materials, and templates.

5. Deploy

The Skill is installed and becomes available to the AI assistant. Future similar tasks are handled automatically.

6. Evolve

As new tasks use the Skill, gaps and improvements are identified. The Skill is updated through a validated change process with strict quality controls.

Key Takeaways

  • Every completed task produces structured documentation (SR-PTD)
  • The documentation format is designed to map directly to reusable Skill components
  • Documents accumulate and are clustered by similarity
  • After 3-5 similar documents, a formal Skill is extracted
  • Skills automate future similar tasks, compounding the team's efficiency
  • Skills evolve through validated updates as new edge cases are discovered
  • The entire process turns one-time problem solving into permanent organizational knowledge
