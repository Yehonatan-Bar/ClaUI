סיכום תוכנית הביצוע: סשן משותף עם סוכני קוד

מטרת הפיצ׳ר

לאפשר למספר מתכנתים ולסוכני קוד שלהם (Claude Code ו-Codex) להשתתף באותה שיחה משותפת. כל מתכנת רואה את כל השיחה בזמן אמת, וכל סוכן מקבל הוראה רק כשמישהו ממוען אליו בדרך ברורה ודטרמינסטית.

ארכיטקטורה בסיסית

שרת תיאום מרכזי - מנהל את השיחה הקנונית, מכתיב לכל סוכן מה עליו לעשות, שומר על סדר ובטיחות
ClaUi Extension - בכל חלון VS Code של משתתף, מנהל את הסוכן המקומי שלו, שולח/מקבל הודעות מהשרת
סוכנים מקומיים - Claude Code ו-Codex רצים בכל משתתף, מפעילים קוד בעבודה משותפת

כיצד זה עובד

  1. משתתף שולח הודעה - אם היא מתחילה בשם או אות של סוכן (למשל "ד: תבדוק את זה"), השרת מנתב אותה אליו
  2. השרת שולח prompt לסוכן - עם קונטקסט של ההודעות החדשות מאז הפעם האחרונה
  3. הסוכן עובד ומחזיר תוצאה - התוצאה מוצגת לכולם
  4. אם סוכן מדבר לסוכן אחר - השרת משעה ומבקש אישור משתתף אנושי (הגנה מפני לולאות אינסופיות)

שיפור מרכזי בהערת המשתמש

בעיה קודמת: כאשר סוכן קודקס עובד וצריך לעדכן אותו למשימה חדשה, היה צריך שלב 1) לשלוח את ההודעה ו-2) ללחוץ על כפתור steer בנפרד.

הפתרון: כל זה אוטומטי עכשיו. השרת שולח prompt אחת עם הsteer flag מובנה, ללא התערבות משתמש נוספת. המשתמש שלח את ההודעה - השאר קורה אוטומטית.

תכונות עיקריות

1. ניתוב חכם

  • ניתן להשתמש בשם מלא ("Claude") או אות אחת ("ק")
  • שפיר לעברית וסימנים אחרים
  • ודא שאין התנגשויות בשמות או באותיות

2. הגנה מלופים בין סוכנים

  • מצב ברירת מחדל: כל פנייה בין סוכנים דורשת אישור (טוב וביטוח מקסימלי)
  • מצב תקציב: אפשר להסכים ל-N הודעות בין סוכנים, אז עצור
  • מצב אוטומטי: הסכם לתקשורת חופשית, אבל משומר "שומר" - דגם קל שבודק כל 20 הודעות אם עדיין קורה משהו יוצר או זה לולאה אינסופית
  • מצב כפוי: לא בטוח - בשימוש רק על ידי משתתפים שיודעים מה הם עושים

3. מודעות לתנגשויות קבצים

  • השרת עוקב אילו קבצים משתנים על ידי איזה סוכן
  • אם שני סוכנים משנים את אותו קובץ בו-זמנית, כל המשתתפים מקבלים אזהרה
  • אפשרות "תכנית בלבד" - סוכנים מתכננים שינויים ללא שינוי בפועל

4. מצביעי פעילות בזמן אמת

  • כל משתתף רואה מיד כאשר מישהו הוא מקלידה או סוכן חושב/מייצר תוצאה
  • זה עוזר להתאמות משותפות ולהבנת מה קורה עכשיו

מודל הנתונים

  • סשן: שיחה משותפת עם הגדרות וכללים
  • משתתף: אדם או סוכן עם שם, אות ניתוב, סטטוס (מקוון/לא מקוון)
  • הודעה: משהו שנאמר בשיחה, קישור למשימה שיגרמה לסוכן לענות
  • מסירה: סוכן מקבל prompt - מעקב אחרי הצלחה, כשל, ביטול
  • בקרת A2A: משטר של אישורים או שומר לשמירה על בטיחות

שלבי הביצוע

שלב 1: סוגים וחוזים (1 יום)

הגדר את כל סוגי ההודעות, מבנים הנתונים, וחוזים עם השרת. יכולים להיות בהקבילה.

שלב 2: ליבה עיקרית (3-5 ימים, בהקבילה)

  • שרת: ניתוב, מנגנון A2A, שירות שומר
  • שרת: שינוי שמות, שמירת נתונים, מצביעי קלדה
  • Extension: עקוב קבצים, חיבור סוכנים
  • Frontend: חנות Zustand, רכיבי React בסיסיים

שלב 3: תכונות מתקדמות (2-3 ימים, בהקבילה)

  • גילוי תנגשויות קבצים, תכנית בלבד
  • מצביעים מתקדמים, קישורים בין הודעות
  • דיאלוג אישור, כפתור עצור

שלב 4: שילוב וביצוח (3-5 ימים, סדרתי)

  • עברה ל-React מלא
  • בדיקה end-to-end עם מתכנתים אמיתיים
  • טיפול בשגיאות, התחברות מחדש, שפה עברית

זמן כולל משוער

10-14 ימים עם מטלה מקביל (לעומת 20-25 ימים אם ברתתי)

סיכום

התוכנית מספקת דרך בטוחה וגמישה למספר מתכנתים לעבוד בצורה משותפת עם סוכני קוד שלהם. הגנה עמידה מלופים משדרת שהתוכנית לעולם לא תלכד בלולאה אינסופית, בעוד שאפשרות "תמיד" עם שומר אומרת שיכול להיות ריצה אוטומטית כשהמשתתפים בטוחים בזה.

השיפור המרכזי: כל ה-steering אוטומטי - אף התערבות משתמש לא נדרשת מעבר לשליחת ההודעה הראשונית.
