תוכנית: ממשק אישור תוכנית (Plan Mode)

Claude Code Mirror - תוספת פיצ'ר חדש

מה הבעיה?

כשקלוד נמצא ב"מצב תוכנית" ויוצר תוכנית עבודה, המערכת מחכה לאישור המשתמש. כרגע, הממשק לא מזהה שקלוד מחכה - והמשתמש רואה רק "קלוד חושב..." לנצח, בלי שום אפשרות לאשר, לדחות, או להגיב.

שורש הבעיה: אירוע קריטי מהמערכת נופל בין הכיסאות ולא מטופל כלל.

מה הפתרון?

זיהוי אוטומטי של הרגע שבו קלוד מציג תוכנית ומחכה לאישור, והצגת פס פעולות ברור למשתמש:

איך ייראה הממשק?

Plan Ready for Review
Approve
Reject
Give Feedback

שלושה כפתורים פשוטים שמחליפים את הודעת "קלוד חושב...":

  • Approve - אישור התוכנית, קלוד ממשיך לעבוד
  • Reject - דחיית התוכנית, קלוד חוזר לתכנן מחדש
  • Give Feedback - שליחת הערות והנחיות לשיפור התוכנית

איך זה עובד?

1
קלוד יוצר תוכנית ומבקש אישור (כלי ExitPlanMode)
2
המערכת מזהה את ההפסקה - הודעת "חושב..." נעלמת
3
מופיע פס אישור עם שלושה כפתורים
4
המשתמש בוחר פעולה (אישור / דחייה / משוב)
5
התגובה נשלחת לקלוד והוא ממשיך לעבוד בהתאם

מה משתנה?

רכיב שינוי היקף
זיהוי אירועים טיפול באירוע "הפסקה לאישור" שנופל כרגע קטן
גשר הודעות זיהוי כלי אישור + העברת הודעות בינוני
סוגי הודעות שני סוגי הודעה חדשים (בקשת אישור + תגובה) קטן
ניהול מצב שדה חדש "ממתין לאישור" במצב האפליקציה קטן
רכיב ממשק חדש פס אישור עם כפתורים ואפשרות משוב בינוני
תצוגת כלי עיצוב מותאם להצגת תוכנית (במקום JSON גולמי) קטן
עיצוב CSS סגנונות לפס האישור והכפתורים קטן

סיכום

שינוי ממוקד ב-9 קבצים שפותר בעיה קריטית בחוויית המשתמש: במקום "קלוד חושב..." שנתקע לנצח, המשתמש רואה את התוכנית ויכול לאשר, לדחות, או לתת משוב - בדיוק כמו בממשק הרגיל של Claude Code.
