Local Boost

Command output compression for AI coding agents. Less tokens in, faster responses out, secrets never leaked.

TL;DR: Local Boost intercepts Bash commands from Claude/Codex, runs them through a local compressor that redacts secrets, filters noise, and returns only the useful output. The agent sees clean, concise results — using fewer tokens, responding faster, and never seeing your API keys.
⚡ Why Local Boost?
2-8x
Output Compression
npm test output of 12,000 lines becomes 80 focused lines. Agent reads less, thinks faster.
~70%
Token Savings
Fewer tokens consumed per command = more headroom in context window = longer, more productive sessions.
0
Secrets Leaked
API keys, tokens, passwords in env vars and command output are redacted before the agent ever sees them.

↓

↓
5
Apply smart output filter

↓
6
Return compressed output + original exit code
Agent receives clean, focused output. Trace file saved for analytics.

⚙ Smart Filters

Each command is matched to a specialized filter. If no specific filter matches, the Generic filter applies.

Filter Matches What It Keeps What It Strips
JS Package npm, pnpm, yarn, bun Errors, test results, audit counts Funding notices, deprecation spam, progress bars
Pytest pytest FAILURES, tracebacks, assertion diffs, summary Collection lines, platform info, dot progress
Jest/Vitest jest, vitest Expected/Received, stack frames, summary PASS lines, watch hints, coverage tables
TypeScript tsc Errors grouped by file (max 10/file) Duplicate diagnostics, verbose paths
ESLint eslint Issues grouped by file (max 10/file) Passing files, verbose rule descriptions
Generic Everything else First 20 + last 80 lines, errors, summaries ANSI codes, spinners, duplicate lines
🔒 Security Layer
1
Environment Variable Scanning
All env vars matching *_TOKEN, *_SECRET, *_KEY, *_PASSWORD etc. are collected. Their values are replaced with [REDACTED] in any output.
2
Pattern-Based Detection
Regex rules catch GitHub PATs (ghp_*), AWS keys (AKIA*), JWTs, OpenAI keys, and other known secret formats — even if not in env vars.
3
Fail-Closed Design
If the redactor encounters an error, the entire output is suppressed. No partial or un-redacted data ever reaches the agent.
4
Network Isolation
claui-run blocks all network access (fetch, http, net, dgram) at the Node.js level. Command output cannot be exfiltrated.
▣ Architecture
Three execution contexts, zero network access in the runner: VS Code Extension Host (Node.js) LocalBoostService → lifecycle, settings, context store LocalBoostInstaller → copies runtime to globalStorage LocalBoostEnvBuilder → injects CLAUI_LOCAL_BOOST_* env vars LocalBoostHookManager → installs/removes pre-tool-use hooks LocalBoostTraceReader → reads traces for dashboard claui-run CLI (Separate Node.js process, NO NETWORK) SecretRedactor → env scanning + regex rules, fail-closed CommandEligibility → allow/deny list, family classification OutputFilterRegistry → 6 specialized + 1 generic filter executeShellCommand → spawn, byte cap, signal forwarding CommandTraceWriter → atomic trace file for analytics Webview (Browser/React) LocalBoostStatusBadge → green/red dot in StatusBar LocalBoostSettingsPanel → toggle, hook install/remove LocalBoostTracePanel → aggregate stats dashboard
☑ Command Eligibility

Eligible (routed through Local Boost)

git status / diff / log / blame
npm test / install / build / lint
pnpm / yarn / bun equivalents
python / pytest / pip
cargo build / test / clippy
go build / test / vet
tsc / eslint / prettier
make / cmake / gradle / mvn
ls / cat / find / grep / wc
docker build / compose

Denied (runs normally, unmodified)

ssh / scp / rsync        # interactive/network
sudo / su               # privilege escalation
vim / nano / emacs      # interactive editors
npm run dev / start     # long-running servers
docker run              # container exec
rm -rf /                # destructive
cmd | cmd               # pipelines (safety)
cmd > file              # redirections
