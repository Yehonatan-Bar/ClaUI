📋 תוכנית שיפור מסנן הסודות (Particle Accelerator)

סיכום בקצרה: אנו משדרגים את תוסף סינון הסודות בקוד כדי להגן על מידע רגיש יותר — סביבה משתנים, הלוואות API, ונתיבים רגישים. המערכת תהיה אוטומטית, שקופה, וכוללת רישום ביקורת מלא לתאימות חוקית.

🎯 מה מקבלים אנחנו?

  • הגנה מתקדמת: 4 סוגי סקנרים שונים שתופסים סודות במיקומים שונים בקוד
  • מדיניות גמישה: להגדיר כללים שונים לסודות שונים (מחזירות, הסתרה, חסימה)
  • ביקורת מלאה: רישום של כל החלטה (מי, מה, מתי, למה, איזה פעולה)
  • ממשק משתמש: תצוגה ברורה של סודות שנמצאו, מעשים שנעשו, והערכות סיכון
  • תאימות חוקית: Git pre-commit hook שמונע הדליפות לרפואה

📊 עובדים בחכמה

התוכנית מחולקת ל-3 שלבים סדרתיים (שכל אחד חוסם את הבא) ו-15 רצועות עבודה מקבילות — זה חוסך זמן.

שלב תיאור משך זמן דברים שמחכים
שלב 1 בנית הבסיס: 6 סקנרים וסך הסודות 3-4 ימים שלב 2, שלב 3
שלב 2 מדיניות וטיוטות סיכון 2-3 ימים שלב 3, ממשק
שלב 3 Git auditing וביקורת 2-3 ימים בדיקה סופית
שלבים 4-5 15 ס\"מ עבודה מקבילה (ממשק, בדיקות, תיעוד) מקבילי, 3-4 ימים —
שלב 6 בדיקה, הפעלה, תקינות 2-3 ימים —
⏱️ לוח זמנים:
  • שלבים 1-3 סדרתיים: 4 + 3 + 3 = 10 ימים
  • שלבים 4-5 מקבילים: חוסכים 4-5 ימים
  • סה\"כ: 9-14 ימים עד להפעלה בייצור

💪 יתרונות

  • ⏚ אבטחה חזקה יותר: מצליחה סודות שלא נשמרה בדיקה קודמת
  • 📋 עד לביקורת: 100% של החלטות מתועדות, מעניקים תאימות חוקית
  • 👁️ שקיפות: המשתמשים רואים בדיוק מה המערכת עשתה ולמה
  • 🔧 גמיש: מדיניות מותאמות אישית לסוגי סודות שונים
  • ✅ ביחזור אחורה: אתרים קיימים עוד עובדים, אין שבירה

⚠️ סיכונים וכיצד אנו מתמודדים

סיכון כמה קשה? פתרון
Git hook חוסם commit-ים שרים בינוני محرك אישור עם whitelist ו approval flow
ה-entropy scan מוצא \"סודות\" שלא באמת גבוה כיוונון סף, הקשר המודעות (סאל בקיר משמחים)
בדיקות תאימות חוקית נמוך בדיקות אחורה נרחב, ללא שינוי קיים API
הפחתת ביצועים נמוך פרופיל משימה, מטרה <50ms מוצבעות עבודה

✅ איך נדע שזה עובד?

  • 14 קבצי בדיקה עם 100+ מקרי בדיקה — כולם עובדים ✓
  • ממשק משתמש מראה סודות שנמצאו וביקורת מלא ✓
  • Git pre-commit תופס סודות לפני commit ✓
  • Backward compatibility: תוספים קודמים עובדים בדיוק כמו קודם ✓
  • ביצועים: כל בדיקה סודות ב<50ms ✓

🔄 מה בתוך הקופסה?

בנייה טכנית (8 קבצי core + 6 קבצי בדיקה בשלב 1):
סקנר המשתנים, סקנר regex, סקנר entropy, סקנר נתיבים, סקנר JSON/YAML, מנוע redaction

מדיניות וסיכונים (3 קבצים בשלב 2):
מנוע מדיניות, מסווג יעדים (API, ענן, גיט), אנליזת סיכון

Git auditing (3 קבצים בשלב 3):
Pre-commit scanner, approval engine, audit logger

ממשק משתמש (5 רכיבים בשלב 5):
תג סטטוס, פנל ביקורת, תצוגת יעדים, הגדרות מדיניות

🎓 סיבות נבחרות עיצוב

  • מודולר: כל סקנר ניתן לעדכון / שינוי בנפרד
  • בדיקות ממצות: כל סקנר וכל מנוע נבדק בנפרד
  • גידול מדיניות: ניתן להוסיף סוגי סודות חדשים ללא שינוי קוד
  • דיוק ביקורת: כל החלטה מתועדת (נדרש חוק)
