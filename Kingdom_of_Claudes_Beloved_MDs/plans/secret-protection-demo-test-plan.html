מטרת התוכנית

הבדיקה תראה מה נחשף כש-Secret Protection כבוי, מה מזוהה במצב תצפית, ומה נחסם, מסונן, מקבל אזהרה או דורש אישור במצבי ההגנה. בנוסף היא תמדוד שעבודת Claude Code רגילה ממשיכה לעבוד ללא חסימות שווא וללא האטה מהותית.

אין לקבוע מראש "100% זיהוי" או "0% חיובי שגוי". אלו יעדי קבלה. הדו"ח הסופי ייבנה מתוך JSON מדוד ויציג הצלחות וכשלונות בפועל.

ארכיטקטורת הבדיקה

1. ספריית סודות מזויפים

Fixture library תחת tests/secret-protection-demo/fixtures/ עם manifest שממפה כל דוגמה ל-rule IDs, סוג ממצא, חומרה, ו-boundaries רלוונטיים.

2. Runner אוטומטי

סקריפט שמריץ מטריצת fixture x boundary x mode, שומר ממצאים, החלטות, תוצר מסונן, audit ו-latency.

3. הדגמה חיה

תרחישי OFF, OBSERVE, BALANCED, STRICT, Browser approval, Git publish, MCP, ו-Business as usual.

4. דו"ח HTML מתוך JSON

הדו"ח הסופי ייווצר מתוך demo-results.json, תצלומי מסך, ולוגי audit. לא מספרים קשיחים בקובץ HTML.

פקודות אימות

בדיקה אוטומטית:

npm run build ואז npm run demo:secret-protection

דמו חי ב-VS Code:

npm run deploy:local ואז npm run verify:installed ולאחר מכן Reload Window.

קריטריוני קבלה

  • כל 13 ה-boundaries מופיעים ב-JSON, כולל סימון scanner-only כשאין caller מחובר.
  • כל rule ID צפוי מזוהה לפחות פעם אחת או נכשל עם סיבה מפורשת.
  • במצבי הגנה אין חשיפה שמנוגדת למדיניות.
  • תוכן נקי לא נחסם ולא מסונן ללא הצדקה.
  • אותו workflow רגיל של Claude Code מסתיים בהצלחה גם עם ההגנה.
  • דו"ח HTML סופי נוצר מתוך JSON מדוד ולא מתוך מספרים קשיחים.
