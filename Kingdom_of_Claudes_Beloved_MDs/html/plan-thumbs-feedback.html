תוכנית: כפתורי פידבק - אגודל למעלה / למטה

הוספת יכולת משוב מהיר על תשובות העוזר, כולל שליחה אוטומטית של לוגים למפתח

מה הפיצ'ר הזה עושה?

כל הודעת תשובה של העוזר (Assistant) תציג שני כפתורים קטנים: אגודל למעלה (תשובה טובה) ו-אגודל למטה (תשובה לא טובה).

  • הכפתורים מופיעים בריחוף עכבר, כמו כפתור ה-Copy וה-Translate הקיימים
  • לחיצה על אגודל למעלה - שולח את הפידבק מיידית
  • לחיצה על אגודל למטה - מציג שדה תגובה קצר (אופציונלי, עד 200 תווים) לפני שליחה
  • אפשר לשנות את הבחירה (ללחוץ על האגודל ההפוך)
  • הפיצ'ר ניתן לכיבוי/הפעלה דרך הגדרות VS Code

מה נשלח למפתח?

כל לחיצת פידבק שולחת למפתח (דרך Formspree - אותו כלי שכבר קיים לדיווחי באגים) את הנתונים הבאים:

נתוןתיאור
הצבעהאגודל למעלה או אגודל למטה
תגובה (אופציונלי)טקסט חופשי קצר שהמשתמש כתב
ההודעה שנבחרהתוכן התשובה של העוזר שקיבלה את ההצבעה
היסטוריית שיחהכל ההודעות מתחילת השיחה עד ההודעה שנבחרה
פרטי סשןשם הסשן, מודל, ספק (Claude/Codex), גרסת התוסף
מידע מערכתמערכת הפעלה, גרסת VS Code, גרסת Node
לוגים אחרוניםכל הלוגים מ-30 הדקות האחרונות (עד 50KB)

שלבי פיתוח

  1. הוספת הגדרה חדשה ב-package.json - הפעלה/כיבוי של הפיצ'ר
  2. הגדרת סוגי הודעות חדשים לתקשורת בין הממשק לתוסף
  3. הוספת ניהול מצב (State) בממשק - מעקב אחרי הצבעות ומצב שליחה
  4. בניית הכפתורים בממשק המשתמש - אייקונים, אנימציות, שדה תגובה
  5. יצירת שירות פידבק חדש - אוסף נתונים ושולח דרך Formspree
  6. חיבור הטיפול בהודעות בצד התוסף - עבור Claude ו-Codex
  7. הוספת עיצוב CSS לכפתורים ולשדה התגובה
  8. בדיקות ואימות מקצה לקצה

נקודות חשובות

  • שימוש בתשתית קיימת - לא מייצרים מנגנון שליחה חדש. הפידבק נשלח דרך Formspree שכבר עובד בדיווחי באגים
  • איסוף לוגים אוטומטי - מערכת האיסוף של DiagnosticsCollector כבר קיימת ובדוקה
  • תמיכה בשני הספקים - הפיצ'ר עובד גם עם Claude וגם עם Codex
  • ניתן לכיבוי - המשתמש יכול לכבות את הכפתורים דרך הגדרות VS Code
  • חוויה חלקה - הכפתורים מופיעים רק בריחוף, לא מפריעים לזרימת העבודה
  • גודל חבילה - השיחה נחתכת ל-30,000 תווים והלוגים ל-50,000 תווים כדי לא לחרוג ממגבלות Formspree
