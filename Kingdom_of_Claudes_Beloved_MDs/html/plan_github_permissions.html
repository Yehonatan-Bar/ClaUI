תוכנית: צמצום הרשאות GitHub לפיצ'ר הקהילה

הבעיה

כשמשתמש מתחבר ל-GitHub דרך פיצ'ר הקהילה, מוצגת לו רשימת הרשאות מוגזמת: שליטה מלאה על ריפוזיטוריז פרטיים, גישה ל-Codespaces, עדכון Workflows ועוד. הסיבה היא שהתוסף משתמש במנגנון ההתחברות המובנה של VS Code, שרשום עם כלל ההרשאות של כל התוספים. בפועל, הפיצ'ר שלנו צריך רק הרשאה אחת: יצירת Gist.

הפתרון

במקום להשתמש במנגנון ההתחברות המובנה של VS Code, המשתמש ייצור Personal Access Token (PAT) ב-GitHub עם הרשאה אחת בלבד: gist. הטוקן יאוחסן בצורה מוצפנת ומאובטחת בתוך VS Code, ויישאר שמור גם אחרי הפעלה מחדש.

חוויית המשתמש החדשה

1
המשתמש לוחץ "התחבר עם טוקן" בפאנל הקהילה
2
לינק נוח מפנה אותו ליצירת טוקן ב-GitHub (עם ההרשאה הנכונה כבר מסומנת)
3
המשתמש מדביק את הטוקן בתיבת קלט מוסתרת (כמו סיסמה)
4
המערכת מוודאת שהטוקן תקין ושיש לו את ההרשאה הנדרשת
5
הטוקן נשמר בצורה מוצפנת - אין צורך להתחבר שוב אחרי הפעלה מחדש

נושא לפני (OAuth) אחרי (PAT)
