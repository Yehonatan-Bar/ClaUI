נושא: הגנה מפני דליפת מפתחות API לקוד צד-לקוח - כלי SPA
מצורף: spa-capability-demo-report.html

שלום,

בעקבות אירוע האבטחה מ-14.05 שבו מפתח Google API דלף מקוד צד-לקוח פומבי וגרם לחיובי הונאה של כ-170,000 שקלים, פיתחתי כלי הגנה שמונע את הסוג הזה של דליפה באופן אוטומטי.

הבעיה שהיתה: מפתחות API היו מוטמעים בקוד HTML/JavaScript שנגיש מהדפדפן. בוטים אוטומטיים סרקו את האתרים, מצאו את המפתחות, וניצלו אותם לקריאות API בתשלום.
הפתרון - Super Particle Accelerator (SPA): הכלי יושב בתוך סביבת הפיתוח (ClaUi) ומיירט כל פעולת כתיבה של סוכן AI. ברגע שהוא מזהה מפתח API, סיסמה, טוקן, או כל סוד אחר - הוא חוסם את הכתיבה לפני שהיא מתבצעת.

מה הכלי עושה בפועל:

  • סורק כל קובץ שסוכן AI מנסה לכתוב או לערוך
  • מזהה מפתחות של Google, OpenAI, AWS, GitHub, Stripe, Slack, Azure ועוד
  • חוסם חסימה קשיחה כל ניסיון לשים סוד בקוד צד-לקוח (public, dist, build, frontend) - ללא אפשרות לעקוף
  • חוסם פקודות git push / commit כשיש סודות בקבצים
  • חוסם שליחת סודות דרך כלים חיצוניים (Slack, email, GitHub Issues)
  • מנחה את הסוכן לתקן: "העבר את הסוד למשתנה סביבה בצד שרת, חשוף proxy endpoint"

תוצאות הניסוי (מצורף):

הרצתי 27 תרחישים עם מפתחות דמה - כולל תרחיש זהה לאירוע שלנו (מפתח Google API בדף HTML ציבורי). הכלי חסם 23 ניסיונות דליפה והעביר 4 פעולות בטוחות. 27 מתוך 27 תרחישים עברו כצפוי.

הדוח המצורף מפרט כל תרחיש, מה נחסם, ולמה.

הערה: הכלי מגן על פעולות של סוכני AI (Claude, Codex). להגנה על מפתחים אנושיים כבר מותקן gitleaks כ-pre-commit hook - יחד זה נותן כיסוי מלא.
