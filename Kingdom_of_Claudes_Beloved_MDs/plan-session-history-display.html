תיקון באג: הצגת היסטוריית שיחה

תיקון - ClaUi

מה הבעיה?

כשמשתמש בוחר שיחה ישנה מהיסטוריית השיחות, נפתח חלון ריק לגמרי. השיחה ממשיכה ברקע (Claude "זוכר" את ההקשר), אבל ההודעות הקודמות לא מוצגות על המסך.

הסיבה: כשחוזרים לשיחה, המערכת מקבלת מ-Claude את כל ההודעות הקודמות. הודעות של המשתמש נשמרות בהצלחה, אבל הודעות של Claude נשמרות רק באופן זמני ולא מתווספות לתצוגה. הן "נופלות" בדרך.

מה הפתרון?

התיקון: כשמגיעה הודעת תשובה שלמה מ-Claude (לא הודעה שעדיין בשידור חי), המערכת תזהה שזו הודעה היסטורית ותוסיף אותה ישירות לתצוגה, במקום לשמור אותה באופן זמני.

איך זה עובד עכשיו?

Session resumed → CLI replays messages → User msgs shown → Assistant msgs → LOST

איך זה יעבוד אחרי התיקון?

Session resumed → CLI replays messages → User msgs shown → Assistant msgs → SHOWN

מה משתנה בקוד?

2 קבצים מושפעים

אזור שינוי
ניהול מצב (Store) פעולה חדשה שמוסיפה הודעת Claude שלמה ישירות לרשימת ההודעות
מטפל הודעות (Webview) זיהוי אוטומטי: הודעה היסטורית -> תצוגה ישירה, הודעה בשידור חי -> התנהגות קיימת

מה לא משתנה?

שום דבר בצד השרת. התיקון הוא רק בצד התצוגה. כל מנגנון השיחה החיה (streaming), ביטול, חידוש אוטומטי, ועריכת הודעות - ממשיכים לעבוד בדיוק כמו קודם.
