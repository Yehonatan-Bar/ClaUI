תיקון תיאור התוסף בחנות VS Code

הבעיה

כרגע, הדף של התוסף בחנות התוספים של VS Code מציג הוראות טכניות למפתחים במקום תיאור למשתמשים:

  • הוראות התקנה מקוד מקור (clone, npm install)
  • תרשימי ארכיטקטורה פנימיים
  • מבנה תיקיות של קוד המקור
  • הגדרות Webpack ופרטי בנייה
  • תיעוד פנימי של רכיבים

משתמש שמחפש תוסף בחנות רואה מידע שלא רלוונטי לו בכלל.

הפתרון

לשכתב את קובץ התיאור (README) כך שיציג מידע רלוונטי למשתמשי קצה, ולהעביר את התיעוד הטכני לקובץ נפרד למפתחים.

שלבי הביצוע

1 שכתוב דף החנות

הדף החדש יכלול:

  • תיאור קצר - מה התוסף עושה
  • רשימת יכולות (מולטי-טאב, בחירת מודל, שליחת קבצים, הדבקת תמונות, ועוד)
  • דרישות מוקדמות (Claude CLI)
  • איך מתחילים - התקנה מהחנות ולחיצה על קיצור מקלדת
  • טבלת קיצורי מקלדת
  • טבלת הגדרות
2 העברת תיעוד טכני לקובץ נפרד

כל המידע הטכני-פנימי יעבור לקובץ חדש בשם DEVELOPMENT.md, שמיועד למפתחים שרוצים לתרום לפרויקט.

3 הגנה על גודל החבילה

הקובץ הטכני לא יארז לתוך התוסף עצמו, כדי לא להגדיל את גודל ההורדה.

השפעה

  • דף התוסף בחנות ייראה מקצועי ורלוונטי למשתמשים
  • שום מידע טכני לא ילך לאיבוד - הוא פשוט עובר למקום מתאים יותר
  • אין שינוי בקוד או בפונקציונליות של התוסף עצמו
  • אין סיכון - מדובר בשינוי תיעוד בלבד
